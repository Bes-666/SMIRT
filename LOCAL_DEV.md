# Локальная разработка SMIRT

## Быстрый старт

### 1. Установите зависимости

```bash
npm install
```

Это установит:
- Netlify CLI (локально в проекте)
- Все необходимые зависимости

### 2. Запустите локальный сервер

```bash
npm start
```

или

```bash
npm run dev
```

### 3. Откройте в браузере

После запуска вы увидите что-то вроде:
```
◈ Netlify Dev ◈
◈ Server now ready on http://localhost:8888
```

Откройте http://localhost:8888 в браузере.

## Что происходит при запуске

- `netlify dev` запускает:
  - Локальный сервер для статических файлов (public/)
  - Netlify Functions локально (netlify/functions/)
  - Проксирует запросы к функциям

## Проверка перед деплоем

1. **Проверьте функциональность:**
   - Отправка сообщений работает
   - Сообщения отображаются
   - Автообновление работает
   - Копирование текста не прерывается

2. **Проверьте дизайн:**
   - Неоновые эффекты работают
   - Анимации плавные
   - Адаптивность на мобильных

3. **Проверьте в разных браузерах:**
   - Chrome/Edge
   - Firefox
   - Safari (если есть Mac)

4. **Проверьте на мобильном:**
   - Откройте http://ваш-ip:8888 на телефоне
   - Или используйте ngrok/tunnelmole для доступа

## Альтернативный способ (без Netlify CLI)

Если не хотите устанавливать Netlify CLI, можно использовать простой HTTP сервер:

```bash
# Установите http-server глобально
npm install -g http-server

# Запустите в папке public
cd public
http-server -p 8888
```

**Внимание:** В этом случае Netlify Functions не будут работать локально, только статические файлы.

## Остановка сервера

Нажмите `Ctrl+C` в терминале, где запущен сервер.

## Полезные команды

```bash
# Проверить версию Netlify CLI
npx netlify --version

# Запустить с дополнительными опциями
netlify dev --port 3000  # Изменить порт

# Запустить с отладкой
DEBUG=* netlify dev
```

## Решение проблем

### Порт занят
Если порт 8888 занят, Netlify автоматически попробует другой порт (8889, 8890 и т.д.)

### Функции не работают
Убедитесь, что:
- Файлы в `netlify/functions/` существуют
- `netlify.toml` настроен правильно
- Нет ошибок в консоли

### Стили не применяются
- Очистите кэш браузера (Ctrl+Shift+R)
- Проверьте консоль браузера на ошибки
- Убедитесь, что файлы сохранены
